# ScriptCraft AI（剧本工坊）— 软件说明书

> **版本**：v1.0.0  
> **定位**：AI 驱动的电影/电视剧本创作一体化平台  
> **核心理念**：一个软件，五大创作模块——不是五个独立的 AI 聊天机器人

---

## 一、设计目的

ScriptCraft AI 致力于解决影视剧本创作中三个核心痛点：

| 痛点 | 传统方式 | ScriptCraft AI 的解法 |
|------|----------|----------------------|
| **创作孤岛** | 灵感、角色、大纲、剧本、对白分散在多个工具（Word / Excel / Miro / Final Draft），创作者反复搬运内容 | 五大模块共享同一份项目数据，AI 自动串联上下文 |
| **AI 碎片化** | 每次对话都是一个孤立的 Chat 窗口，AI 不知道你的故事全貌 | 每个模块有独立的 system prompt，但共享完整的项目上下文（梗概、角色、大纲） |
| **技术门槛** | 剧本软件要么太简陋（记事本），要么太复杂（Final Draft 上手成本高） | 浏览器即用，零安装，AI 驱动内容生产，创作者专注「创作」本身 |

**一句话总结**：让创作者用「对话」的方式完成从灵感到剧本的全流程，AI 不是替代创作者，而是 24 小时在线的创作伙伴。

## 二、设计思路

### 2.1 产品架构：一个软件，五个模式

```
ScriptCraft AI
│
├── 灵感工坊 ─── 故事构思 · 主题探索 · Logline
├── 角色档案 ─── 角色创建 · 性格弧光 · 关系图谱
├── 分场大纲 ─── 节拍表 · 三幕结构 · 分场拆解
├── 剧本正文 ─── 标准格式剧本撰写
└── 对白润色 ─── 台词优化 · 角色声音 · 潜台词
```

五个模块共享**同一份项目数据**，切换模块时 AI 自动注入完整上下文。

### 2.2 技术架构

```
┌──────────────────────────────────────────────────┐
│                   用户浏览器                       │
│  ┌─────────┐  ┌──────────┐  ┌──────────────────┐ │
│  │  React  │  │ Tailwind │  │  Zustand Store    │ │
│  │   18    │  │   CSS    │  │  (状态管理)        │ │
│  └─────────┘  └──────────┘  └──────────────────┘ │
│        │              │              │            │
│        └──────────────┼──────────────┘            │
│                       │                           │
│              VITE VITE_API_BASE_URL                │
│                       │                           │
└───────────────────────┼───────────────────────────┘
                        │
              ┌─────────▼──────────┐
              │   Express 4 后端    │
              │   (Node.js + TS)   │
              │   SSE 流式响应      │
              └─────────┬──────────┘
                        │
              ┌─────────▼──────────┐
              │     数据库          │
              │  本地: JSON 文件    │
              │  线上: MongoDB     │
              └────────────────────┘
```

### 2.3 关键设计决策

| 决策 | 说明 |
|------|------|
| **AI 密钥归用户** | 用户自带 DeepSeek / 豆包 / 通义千问 API 密钥，存于浏览器 localStorage，不经过服务端持久化 |
| **SSE 流式响应** | AI 回复逐字呈现，类 ChatGPT 体验 |
| **模块化 System Prompt** | 每个模块有专属的 AI 人设（故事顾问 / 角色塑造师 / 结构分析师 / 剧本作家 / 对白导演） |
| **跨模块上下文联动** | AI 请求自动附带项目的完整数据（梗概 + 角色 + 大纲 + 剧本），模块间信息不割裂 |
| **前后端分离部署** | 前端 Vercel（免费）+ 后端 Railway（免费额度）+ MongoDB Atlas（免费），零成本部署 |

## 三、工作流说明

### 3.1 完整创作流程

```
① 创建项目 ──→ ② 灵感工坊 ──→ ③ 角色档案 ──→ ④ 分场大纲 ──→ ⑤ 剧本正文 ──→ ⑥ 对白润色
   │               │              │              │              │              │
   ▼               ▼              ▼              ▼              ▼              ▼
 标题/类型    Logline/梗概    人物小传       三幕结构      分场剧本      台词精修
 一句话梗概   主题探索        性格弧光       分场拆解      动作描写      角色声音
             故事方向        关系图谱       AI生图        场景过渡      潜台词
                                               AI生视频
                                               AI配音
```

### 3.2 各模块工作流详解

**模块一：灵感工坊**
1. 输入模糊想法（一个画面、一段情绪、一个人物）
2. AI 引导头脑风暴，提供多个故事方向
3. 确定核心故事线，生成一句话梗概（Logline）
4. 输出：类型定位、目标受众、市场差异化分析

**模块二：角色档案**
1. 基于 Logline 需求创建角色
2. AI 输出结构化档案：姓名 / 年龄 / 核心欲望 / 致命缺陷 / 弧光起点→终点
3. 可创建多个角色，AI 自动维护角色间的关系网络

**模块三：分场大纲**
1. AI 分析故事结构，生成三幕式节拍表
2. 逐场拆解：场次序号 / 场景 / 时间 / 核心冲突
3. 支持 AI 生图：为每场戏生成概念画面（分镜图）
4. 支持 AI 生视频：为关键场景生成动态预览
5. 支持 AI 配音：为场景角色配音

**模块四：剧本正文**
1. 基于大纲逐场撰写标准格式剧本
2. AI 输出：场景标头 + 动作描写 + 人物对话 + 括注
3. 保持叙事节奏和张弛有度

**模块五：对白润色**
1. 选定已有对白段落
2. AI 分析：角色声音是否一致、潜台词是否丰富
3. 提供多个润色版本供选择

### 3.3 上下文流转示例

```
用户在「角色档案」创建了侦探角色「林默」（性格：内向+强迫症）

→ 切换到「分场大纲」
  AI 自动知道：林默是主角，在结构时以他为主线

→ 切换到「剧本正文」
  AI 写林默的台词时，自然带出强迫症特质（反复确认细节）
```

## 四、数据库配置说明

### 4.1 存储方案

项目采用**双模式自动切换**的存储策略：

| 环境 | 存储方式 | 数据位置 |
|------|----------|----------|
| **本地开发** | JSON 文件 | `screenplay-ai/data/` 目录 |
| **生产部署** | MongoDB Atlas | MongoDB 云端集群 |

切换依据：检测环境变量 `MONGODB_URI` 是否存在。
- **存在** → 自动连接 MongoDB
- **不存在** → 自动使用本地 JSON

### 4.2 数据模型

```
projects（项目）
├── id: 项目唯一标识
├── title: 电影/剧集名称
├── genre: 类型（悬疑/爱情/科幻...）
├── logline: 一句话梗概
└── created_at / updated_at

module_content（模块内容）
├── project_id: 所属项目
├── module_type: ideation | character | outline | screenplay | dialogue
├── content: 模块输出内容（Markdown 文本）
└── metadata: JSON 扩展字段

chat_messages（聊天记录）
├── project_id: 所属项目
├── module_type: 所属模块
├── role: user | assistant
├── content: 消息内容
└── timestamp: 时间戳

media（媒体文件）
├── project_id: 所属项目
├── scene_num: 分场编号
├── type: image | video
├── file_path: 文件名
├── prompt: 生成时使用的提示词
├── provider: 生图/生视频提供商
└── metadata: JSON 扩展字段
```

### 4.3 本地开发（默认）

无需任何配置，启动后自动在 `data/` 目录创建 JSON 文件：
```
data/
├── projects.json
├── module_content.json
├── chat_messages.json
├── media.json
└── media/        ← AI 生成的图片/视频文件
```

### 4.4 生产部署（MongoDB Atlas）

在 Railway 控制台设置环境变量：

```
MONGODB_URI=mongodb+srv://scriptcraft-admin:<密码>@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority
```

部署后 MongoDB Atlas 中会自动创建集合：
- `projects`
- `module_content`
- `chat_messages`
- `media`

> ⚠️ 生产环境下媒体文件保存在 Railway 容器本地，重启会丢失。建议后续接入 Cloudinary（免费 25GB）做持久化存储。

## 五、功能说明

### 5.1 核心功能矩阵

| 功能 | 描述 |
|------|------|
| **多项目管理** | 创建 / 删除 / 切换电影/剧集项目 |
| **AI 对话创作** | 五个模块各配备专属 AI，SSE 流式输出 |
| **AI 生图** | 为分镜场景生成概念图，支持即梦AI / Stable Diffusion / 自定义API |
| **AI 生视频** | 为关键场景生成动态预览，支持即梦AI / Runway / Pika / 自定义API |
| **AI 配音** | 为角色配音，文本转语音 |
| **自定义 API** | 支持用户接入第三方 API 中转平台（三个 Tab 均支持自定义 Base URL / Key / Model） |
| **自定义背景** | 上传本地图片作为软件背景，本地 localStorage 持久化 |
| **密钥安全** | API 密钥仅存浏览器 localStorage，通过请求头传后端，服务端不持久化 |

### 5.2 AI 引擎

| 功能 | 支持提供商 | 格式兼容 |
|------|-----------|----------|
| **文本对话** | DeepSeek / 豆包(火山引擎) / 通义千问 / 自定义 | OpenAI Chat Completions |
| **生图** | 即梦AI / Stable Diffusion / 自定义 | OpenAI Images API |
| **生视频** | 即梦AI / Runway / Pika / 自定义 | 任务提交+轮询 |

### 5.3 分镜视觉功能

在「分场大纲」模块，每场戏支持：
- **缩进行信息**：画面描述 / 景别 / 运镜 / 光影 / 色调 / 风格
- **AI 生图**：将分镜数据转为概念图，支持 8 种风格预设（cinematic / realistic / anime / noir / watercolor / oil / scifi / fantasy）
- **AI 生视频**：5-10 秒动态预览，支持单图动效和多镜连贯两种模式
- **一键批量生图**：为所有分场批量生成图片

### 5.4 设置面板

左侧栏底部 ⚙️ 入口，三个标签页：

| 标签 | 配置项 |
|------|--------|
| **AI 文本** | 提供商 / API Base URL / API Key / 模型 |
| **AI 生图** | 提供商 / API Base URL / API Key / 图片尺寸 |
| **AI 生视频** | 提供商 / API Base URL / API Key / 视频时长 |
